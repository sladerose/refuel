--- ADD CommunitySyncSettingsRow view BELOW ProfileView (before the Rank extension) ---

Insert `CommunitySyncSettingsRow` as a private struct within the same file. It must handle:
- Toggle ON: call `socialSyncManager.enableSharing(for:contributionCount:)`, set `isCommunityShareEnabled = true`, show inline "Sharing enabled"
- Toggle OFF: present `.confirmationDialog` with destructive "Stop Sharing" / cancel "Keep Sharing". On confirm: call `socialSyncManager.disableSharing(for:)`, show inline "Sharing disabled"
- CloudKit error: show inline error text, revert toggle

```swift
// MARK: - CommunitySyncSettingsRow (UI-SPEC Component Inventory)

struct CommunitySyncSettingsRow: View {
    @Environment(SocialSyncManager.self) private var socialSyncManager
    @Environment(GamificationManager.self) private var gamificationManager

    @State private var showOptOutDialog = false
    @State private var inlineMessage: String? = nil

    var body: some View {
        VStack(alignment: .leading, spacing: 4) {
            HStack {
                Label("Community Sharing", systemImage: "globe")   // UI-SPEC Copywriting

                Spacer()

                // Opt-in toggle (D-07)
                Toggle("", isOn: Binding(
                    get: { socialSyncManager.isCommunityShareEnabled },
                    set: { newValue in
                        if newValue {
                            socialSyncManager.isCommunityShareEnabled = true
                            Task { await handleToggleOn() }
                        } else {
                            showOptOutDialog = true
                        }
                    }
                ))
                .labelsHidden()
                .tint(.orange)                                      // UI-SPEC accent for toggle thumb
            }
            .frame(minHeight: 44)                                   // UI-SPEC: 44pt minimum touch target

            // Caption description (always visible)
            Text("Share your alias and XP with the global Fuel Scout community.")   // UI-SPEC Copywriting
                .font(.caption)
                .foregroundStyle(.secondary)

            // Inline feedback (toggle state changes and errors)
            if let message = inlineMessage {
                Text(message)
                    .font(.caption)
                    .foregroundStyle(
                        message.hasPrefix("Couldn't") ? Color.red : Color.secondary
                    )
                    .transition(.opacity)
            }

            // Syncing indicator (post-opt-in write in flight)
            if case .syncing = socialSyncManager.syncState, socialSyncManager.isCommunityShareEnabled {
                HStack(spacing: 4) {
                    ProgressView()
                        .controlSize(.mini)
                    Text("Syncing...")
                        .font(.caption)
                        .foregroundStyle(.secondary)
                }
            }
        }
        .animation(.easeInOut(duration: 0.2), value: inlineMessage)
        .animation(.easeInOut(duration: 0.2), value: socialSyncManager.syncState == .idle)
        .confirmationDialog(
            "Stop sharing your stats?",                            // UI-SPEC Copywriting
            isPresented: $showOptOutDialog,
            titleVisibility: .visible
        ) {
            Button("Stop Sharing", role: .destructive) {           // UI-SPEC Copywriting
                Task { await handleToggleOff() }
            }
            Button("Keep Sharing", role: .cancel) {}               // UI-SPEC Copywriting
        } message: {
            Text("Your alias and XP will be removed from the global leaderboard.")  // UI-SPEC Copywriting
        }
        .onChange(of: socialSyncManager.syncState) { _, newState in
            if case .error(let msg) = newState {
                // Revert toggle on error (UI-SPEC Interaction Contract)
                socialSyncManager.isCommunityShareEnabled = false
                withAnimation { inlineMessage = msg }
                // Auto-clear error after 4 seconds
                Task {
                    try? await Task.sleep(for: .seconds(4))
                    withAnimation { inlineMessage = nil }
                }
            }
        }
    }

    // MARK: - Actions

    @MainActor
    private func handleToggleOn() async {
        guard let profile = gamificationManager.userProfile else { return }
        let contributions = gamificationManager.totalContributionCount()
        await socialSyncManager.enableSharing(for: profile, contributionCount: contributions)
        // Show success or error inline
        if case .error = socialSyncManager.syncState {
            // onChange handler above will set inlineMessage from error
        } else {
            withAnimation { inlineMessage = "Sharing enabled" }    // UI-SPEC Copywriting
            Task {
                try? await Task.sleep(for: .seconds(3))
                withAnimation { inlineMessage = nil }
            }
        }
    }

    @MainActor
    private func handleToggleOff() async {
        guard let profile = gamificationManager.userProfile else { return }
        await socialSyncManager.disableSharing(for: profile)
        withAnimation { inlineMessage = "Sharing disabled" }       // UI-SPEC Copywriting
        Task {
            try? await Task.sleep(for: .seconds(3))
            withAnimation { inlineMessage = nil }
        }
    }
}
```

IMPORTANT: `gamificationManager.totalContributionCount()` was added to GamificationManager in Plan 01 as a `@MainActor private func`. It must be made `internal` (remove `private`) so CommunitySyncSettingsRow can access it, OR the call can be made through the profile's existing data. If it's still `private` after Plan 01, change it to `internal` (no access modifier needed — Swift default) in GamificationManager.swift.

Add `import CloudKit` is NOT needed in ProfileView.swift — SocialSyncManager handles all CloudKit interaction.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User toggle → CloudKit write | Toggle ON is the gate for any public data leaving the device; must require explicit user confirmation |
| Toggle OFF → CloudKit delete | Opt-out must reliably delete the public record per D-08 |
| CommunitySyncSettingsRow → SocialSyncManager | UI calls write operations; accountStatus check is in SocialSyncManager not the view |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Information Disclosure | CommunitySyncSettingsRow opt-in | mitigate | Toggle is OFF by default (D-07). No public data written until user explicitly enables. Confirmed via UserDefaults.standard.bool returning false for unset key. |
| T-12-03-02 | Repudiation | Opt-out delete confirmation dialog | mitigate | `.confirmationDialog` with destructive "Stop Sharing" button requires explicit user interaction before `disableSharing` is called. System sheet prevents accidental deletion. |
| T-12-03-03 | Elevation of Privilege | NavigationLink to LeaderboardView | accept | LeaderboardView is a read-only display. Any authenticated user can view the public leaderboard. No privilege escalation possible from the navigation entry. |
| T-12-03-04 | Tampering | Toggle state desync (UI says off, UserDefaults says on) | mitigate | Toggle bound to `socialSyncManager.isCommunityShareEnabled` which reads directly from UserDefaults on every get. Error handler explicitly sets `isCommunityShareEnabled = false` to revert. Single source of truth. |
</threat_model>
