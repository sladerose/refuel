## Spacing Scale

SwiftUI uses point-based layout. All spacing follows the 8-point iOS grid, with 4pt for tight inline gaps.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gaps, inline badge padding (e.g. rank badge pill) |
| sm | 8pt | Intra-row element spacing, vertical stat stack gaps |
| md | 16pt | Default section padding, List row inset (system default) |
| lg | 24pt | Between leaderboard section header and first row |
| xl | 32pt | Between leaderboard and adjacent ProfileView sections |
| 2xl | 48pt | Major content breaks (e.g. empty state illustration to text) |
| 3xl | 64pt | Not used in this phase |

Exceptions:
- Touch targets for opt-in toggle and leaderboard action rows: minimum 44pt height (Apple HIG minimum tappable area).
- "You" highlight row: no extra vertical padding. Visual distinction is provided solely by the `.orange.opacity(0.12)` background overlay — no non-standard spacing values.

## Typography

All sizes expressed as Apple system text styles (Dynamic Type compliant). No fixed point sizes. Source: CONVENTIONS.md mandate.

| Role | System Style | Weight | Line Height |
|------|-------------|--------|-------------|
| Leaderboard section title ("Global Leaderboard") | `.title2` | Semibold (`.semibold()`) | System default (approx 1.2) |
| Rank position + alias (leaderboard row primary) | `.headline` | Semibold (`.semibold()`) | System default (approx 1.35) |
| XP value in leaderboard row | `.headline` + `.monospaced` | Regular | System default |
| Supporting body text (privacy description, opt-in explanation) | `.body` | Regular | System default (approx 1.5) |
| "You" pill / position badge | `.caption` | Semibold (`.semibold()`) | System default (approx 1.2) |
| Rank label (e.g. "Expert Scout"), section header labels, metadata / secondary (e.g. contribution count) | `.caption` | Semibold (`.semibold()`) for section headers and rank labels; Regular for metadata | System default |

Design variants in use:
- `.monospaced` on XP values to prevent layout shift as numbers change.
- `.rounded` design variant on rank badges and the "You" pill for visual warmth consistent with the existing gamification UI.

## Color

This project uses semantic system colors and the established app accent (orange). No hex values are hardcoded — all colors reference `Color` system values or the existing RAG palette from Models.swift.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `.systemBackground` / `.secondarySystemBackground` | Full-screen backgrounds, NavigationStack root |
| Secondary (30%) | `.ultraThinMaterial` (Liquid Glass) | Leaderboard row cards, the highlighted "You" row background |
| Accent (10%) | `.orange` | Accent reserved for: (1) "You" row highlight border/background tint, (2) XP value text on the local user's own leaderboard row, (3) the community-sharing opt-in toggle thumb |
| Semantic — rank badge tints | `.orange` (newcomer) → gradient to `.purple` (fuelLegend) | Rank icon circle backgrounds on leaderboard rows — matches existing ProfileView gradient |
| Destructive | `.red` | Opt-out confirmation only ("Stop Sharing" alert destructive button) |

Accent reserved for: "You" row highlight, local user's XP text, opt-in toggle thumb. NOT used for general row accents, borders, or rank badges of other users.

## Interaction Contract

### Leaderboard View

| Interaction | Behaviour |
|-------------|-----------|
| Pull to refresh | Fetches latest public CKRecords; shows `ProgressView` in place of list during initial load |
| Tap a row (not own row) | No action — rows are read-only display |
| Tap own row ("You") | No action — already in context |
| Scroll past visible "You" row | "You" sticky footer appears at bottom of screen showing user's position and XP |
| Scroll back to own row | Sticky footer dismisses |
| Community sharing OFF | Leaderboard shows read-only data; opt-in prompt banner at top of leaderboard ("Join the leaderboard — enable Community Sharing in Profile > Settings") |

### Opt-In Toggle (Settings section of ProfileView)

| Interaction | Behaviour |
|-------------|-----------|
| Toggle ON (default: off) | Immediately trigger first CloudKit public DB write; show brief inline status "Sharing enabled" using `.caption` text below toggle |
| Toggle OFF (when previously on) | Present `.confirmationDialog` (not full Alert): "Stop sharing your stats?" with destructive button "Stop Sharing" and cancel button "Keep Sharing". On confirm: delete public CKRecord, show inline "Sharing disabled" |
| CloudKit error on write | Show inline `.caption` error text below toggle: "Couldn't connect. Your stats are safe — will retry." Toggle reverts to previous state |

### Empty Leaderboard State

When no community members have opted in yet (zero public records):

- Show SF Symbol: `person.3.fill` at `.largeTitle` scale, tinted `.secondary`
- Heading: "No scouts on the board yet" (`.headline`)
- Body: "Be the first — enable Community Sharing in Profile > Settings." (`.body`, `.secondary`)
- No CTA button — user navigates back to Profile to act

### Loading State

- Initial fetch: full-screen `ProgressView("Loading leaderboard...")` centred in the view
- Background refresh (pull-to-refresh): system `.refreshable` scroll view spinner
- Post-opt-in sync: inline `ProgressView` next to toggle label, width-constrained to row

## State Inventory

All states the leaderboard and opt-in UI must handle:

| State | View | Trigger |
|-------|------|---------|
| Loading (initial) | LeaderboardView | View appears, fetch in flight |
| Loaded — with data | LeaderboardView | Fetch returns 1+ records |
| Loaded — empty | LeaderboardView | Fetch returns 0 records |
| Error — network/CloudKit | LeaderboardView | CKError thrown during fetch |
| Sharing disabled (default) | LeaderboardView + ProfileView | `communitySharing` = false |
| Sharing enabled | LeaderboardView + ProfileView | `communitySharing` = true |
| Opt-in write pending | CommunitySyncSettingsRow | Toggle flipped ON, write in flight |
| Opt-out delete pending | CommunitySyncSettingsRow | Confirmation confirmed, delete in flight |
| "You" row visible | LeaderboardView | User's own row within viewport |
| "You" row off-screen | LeaderboardView | User scrolled past own row — sticky footer active |
