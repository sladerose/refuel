# Phase 2: Price Integration & Persistence - Plan 02-02

## Objective
Implement the background data ingestion layer, the station discovery list with sorting, and update the map with price markers.

Purpose: To provide users with up-to-date fuel price information and the ability to compare stations by cost and proximity.
Output: Service layer, background actor, and discovery UI components.

## Execution Context
- SwiftUI 6.0, SwiftData, MapKit.
- `LocationManager` is already implemented.
- `Station` and `FuelPrice` models are already defined.

## Tasks

### Task 1: Implement Data Ingestion Layer
**Files**: `refuel/FuelPriceService.swift`, `refuel/FuelPriceIngestor.swift`
**Action**:
1. Create `FuelPriceService.swift`:
   - Define `FuelPriceService` protocol with `fetchNearbyStations(latitude:longitude:) async throws -> [StationDTO]`.
   - Implement `MockFuelPriceService` providing stubbed data for at least 3 stations with multiple fuel grades.
2. Create `FuelPriceIngestor.swift`:
   - Define `@ModelActor actor FuelPriceIngestor`.
   - Implement `updatePrices(latitude:longitude:)` to fetch from service and upsert into SwiftData.
   - Logic: Check for existing station by coordinate (or UUID/ID if provided), update attributes, and refresh/replace price relationships.

### Task 2: Create StationListView with Sorting
**Files**: `refuel/StationListView.swift`
**Action**:
Create `StationListView.swift`:
- Use `@Query` to fetch `Station` models.
- Implement a Picker for sorting: "Distance" vs "Price".
- Sorting logic:
  - Distance: Use distance from `LocationManager`.
  - Price: Find the minimum price for a preferred grade.
- Display station name, distance, and current prices for all available grades (INFO-01).

### Task 3: Update MapView and Implement TTL
**Files**: `refuel/MapView.swift`, `refuel/Models.swift`, `refuel/ContentView.swift`
**Action**:
1. Update `Models.swift`:
   - Add `isStale: Bool` computed property to `Station` (e.g., `lastUpdated` > 4 hours ago).
2. Update `MapView.swift`:
   - Query `Station` models and display them as `Annotation` or `Marker`.
   - Markers should display the price (e.g., "$1.95").
3. Update `ContentView.swift`:
   - Use a `TabView` or `NavigationStack` to toggle between `MapView` and `StationListView`.
   - Trigger `FuelPriceIngestor.updatePrices` on view appear or location update if data is stale.

## Verification & Testing
1. Verify `FuelPriceIngestor` can be initialized and called.
2. Verify `StationListView` displays data and sorting changes the order.
3. Verify `MapView` shows markers for persisted stations.
4. Success criteria: The app fetches and persists stubbed fuel prices in the background, users can switch between a map and a list, and stations can be sorted by distance and price.
